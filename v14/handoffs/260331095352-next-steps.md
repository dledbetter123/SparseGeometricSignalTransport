## Hybrid Architecture Options

### Option 1: Wilson Fiber + MLP (Simplest Fix)

**What**: Keep V14's Wilson fiber, restore a small MLP for per-token nonlinearity. Drop Langevin and proximal sparsity (proven dormant/unproven).

**Architecture**:
```
V15Block:
  normed = MagPhaseNorm(constellation)
  messages = WilsonFiber(normed)                    # content-dependent complex EMA
  delta = MLP(normed.mag, normed.phase, messages)   # per-token nonlinear update
  return constellation + gate · delta
```

**Parameter budget per block (~200K)**:
- WilsonFiber: ~105K (wilson_proj MLP + base_decay)
- Update MLP: ~95K (Linear(3×136, 192) → SiLU → Dropout → Linear(192, 2×136))
- Norm + gate: ~0.3K

**Total model**: ~1.7M (8 blocks + embedding + decoder)

**What it tests**: Does Wilson fiber + MLP > plain SSM + MLP (V12.1)? This isolates the Wilson line's contribution from the Langevin/sparsity noise. If Wilson+MLP beats V12.1 (2.30 BPC), the complex phase rotation is a genuine contribution. If not, the Wilson line is decorative and the content-dependent gating alone (which Mamba already does) is all that matters.

**Implementation effort**: 2-3 hours. Mostly simplifying V14 (remove LangevinSettler, add MLP back).

**Risk**: Low. Floor = V12.1 performance. Can't make things worse.

**Best for**: Immediate next experiment. Answers the most important question with minimal effort.

### Option 2: Fiber + Windowed Attention

**What**: Two parallel cross-token paths — Wilson fiber for global geometric context, windowed attention for precise local retrieval. MLP for per-token processing.

**Architecture**:
```
V15Block:
  normed = MagPhaseNorm(constellation)

  # Path 1: Global geometric context [O(n)]
  messages = WilsonFiber(normed)

  # Path 2: Precise local retrieval [O(w²), w=64]
  x_spatial = irfft(normed.to_complex())              # spectral → spatial for attention
  local = WindowedAttention(x_spatial, window=64)      # standard causal windowed attention
  local_spectral = rfft(local)                         # spatial → spectral

  # Combine + nonlinear update
  combined = cat(normed.mag, normed.phase, messages, local_spectral)
  delta = MLP(combined)
  return constellation + gate · delta
```

**Parameter budget per block (~280K)**:
- WilsonFiber: ~105K
- WindowedAttention: ~66K (QKV proj for 256-dim, 4 heads)
- MLP: ~110K (larger to handle combined input)
- Norm + gate: ~0.3K

**Total model**: ~2.4M (matches GPT-Nano)

**What it tests**: Whether the fiber's long-range geometric path adds value that windowed attention alone misses. The fiber accumulates global context through content-dependent phase rotation; the window attention handles precise "which character follows which" patterns. Each does what it's good at.

**Implementation effort**: 1 day. Need to add windowed attention, handle spectral↔spatial conversion.

**Risk**: Medium. The window attention might dominate and the fiber might be unused. But the attention provides a guaranteed performance floor.

**Best for**: Competitive performance. If tuned well, this should match or beat GPT-Nano by combining the best of both paradigms.

**Scaling story**: At long sequences (4K+), full attention is O(n²) but windowed attention is O(n·w). The fiber provides the missing global context at O(n). This is a genuine architectural advantage over full attention transformers.

### Option 3: Parseval Attention (Spectral QK Similarity)

**What**: Use the constellation's spectral geometry AS the attention similarity metric. No learned QK projections — the Parseval inner product between spectral representations IS the attention score.

**Architecture**:
```
V15Block:
  normed = MagPhaseNorm(constellation)

  # Parseval attention: spectral geometry = attention scores
  c_complex = normed.to_complex()                       # (B, T, M) complex
  # Per-subbundle attention (8 independent attention ops, like heads)
  for each subbundle k:
    Q_k = c_complex[..., k*17:(k+1)*17]                 # (B, T, 17) complex
    K_k = c_complex[..., k*17:(k+1)*17]                 # (B, T, 17) complex
    scores_k = Re(Q_k @ K_k.conj().T) / sqrt(17)        # (B, T, T) Parseval similarity
    scores_k = causal_mask(scores_k)
    V_k = linear_k(normed.to_flat())                     # learned value projection
    attended_k = softmax(scores_k) @ V_k

  attended = cat(all subbundle outputs)
  delta = MLP(attended)
  return constellation + gate · delta
```

**Parameter budget per block (~180K)**:
- Parseval attention: ~35K (only V projections + output proj, NO QK projections)
- MLP: ~145K
- Norm + gate: ~0.3K

**Total model**: ~1.5M

**What it tests**: Whether geometric similarity (magnitude overlap × phase alignment) is a useful attention prior compared to learned QK projections. The Parseval inner product provides three things standard attention doesn't:
1. **Phase sensitivity**: Two tokens with same magnitude but different phase score differently
2. **Spectral locality**: Attention within subbundles provides frequency-band-specific routing
3. **No QK parameters**: The metric comes from the representation structure, not learned weights

**Implementation effort**: 1 day. Straightforward but need careful handling of complex attention.

**Risk**: High. The Parseval metric might not capture the right notion of "should attend." Character-level Shakespeare might not have enough spectral structure to make this useful. But if it works, it's a genuinely novel contribution.

**Potential problem**: This is O(n²) in sequence length (full attention). Loses the O(n) advantage of the fiber. Could be combined with windowed version for efficiency.

**Best for**: Theoretical contribution / publication. "Attention where the similarity metric is derived from spectral geometry of the representation space" is a novel claim.

### Option 4: Fiber-Augmented Transformer

**What**: Standard small transformer + Wilson fiber as a parallel side channel. The transformer does what it does; the fiber provides additional geometric context.

**Architecture**:
```
# Main path: standard transformer
x = x + MultiHeadAttention(LayerNorm(x))
x = x + FFN(LayerNorm(x))

# Fiber side channel: parallel O(n) path
constellation = SpectralProject(x)        # learned projection to spectral domain
messages = WilsonFiber(constellation)
fiber_ctx = SpatialProject(messages)      # back to spatial domain

# Fusion
x = x + gate · fiber_ctx                 # fiber injects into residual stream
```

**Parameter budget per block (~250K)**:
- Standard attention + FFN: ~160K (4 heads, 128 dim, 512 FFN)
- Spectral projection: ~35K (Linear 128→272 + Linear 272→128)
- WilsonFiber: ~105K (reuse V14's)
- Gate: learned scalar

**Total model**: ~2.4M

**What it tests**: Whether geometric context accumulation provides signal that standard attention doesn't already capture. This is the safest design: the transformer guarantees a performance floor (GPT-Nano level), and the fiber can only help.

**Implementation effort**: 1 day.

**Risk**: Very low. Floor = GPT-Nano performance. Ceiling = GPT-Nano + whatever the fiber adds.

**Best for**: Proving the fiber has value to a skeptical audience. If the fiber-augmented transformer beats GPT-Nano, you can't dismiss it as "weird architecture artifact" — the improvement is directly attributable to the fiber.

**Downside**: Doesn't test the spectral sparsity or constellation representation. The transformer uses standard dense embeddings; the fiber is a side channel, not the core representation.

### Option 5: Complex Mamba-2 (Linear Attention Framing)

**What**: Mamba-2 showed SSMs and linear attention are mathematically equivalent. Reframe the Wilson fiber in that language: complex-valued gated linear attention.

**Architecture**:
```
V15Block:
  normed = MagPhaseNorm(constellation)

  # Complex gated linear attention (= Wilson fiber, reframed)
  Q = linear_q(normed.to_flat())          # complex query
  K = linear_k(normed.to_flat())          # complex key
  V = linear_v(normed.to_flat())          # value
  decay = wilson_proj(normed.to_flat())   # content-dependent complex decay

  # Recurrence (this IS the Wilson fiber):
  S[t] = decay[t] · S[t-1] + outer(K[t], V[t])   # complex state matrix
  output[t] = Q[t] @ S[t]                          # query against accumulated state

  # Per-token MLP
  delta = MLP(output)
  return constellation + gate · delta
```

**Parameter budget per block (~210K)**:
- Q, K, V projections: ~75K (3 × Linear(272, 136))
- Wilson proj (decay): ~75K (from V14)
- MLP: ~60K
- Norm + gate: ~0.3K

**Total model**: ~1.8M

**What it tests**: Whether complex-valued state matrices (with phase rotation) provide benefit over real-valued states in the gated linear attention framework. This is the most publishable framing: it slots directly into the Mamba-2 / linear attention / state space model literature with a clear, testable hypothesis.

**Implementation effort**: 1.5 days. Need to implement the matrix-valued recurrence and ensure numerical stability.

**Risk**: Low-medium. Well-understood framework (Mamba-2), novel twist (complex states with phase). The linear attention formulation might expose inefficiencies in the current Wilson fiber design.

**Key insight**: The Wilson fiber's parallel scan is already computing gated linear attention with 1D states. Upgrading to a matrix-valued state (outer product K×V instead of scalar deposit) dramatically increases the recurrence's capacity — each mode carries a matrix, not a scalar.

**Best for**: Publication. "Complex-valued gated linear attention with phase rotation outperforms real-valued variants" is a precise, testable, publishable claim.

## Evaluation Setup Changes (Required for Any Hybrid)

### Move to BPE + Larger Corpus

**Current setup**: Character-level, vocab 65, Tiny Shakespeare (1.1M chars)
**Required setup**: BPE tokenization, vocab 4K-8K, WikiText-103 (100M tokens) or OpenWebText subset

**Why**:
- Vocab 4K-8K creates genuine compression pressure for 136 spectral modes (need sparsity)
- 100M tokens gives enough data for 2M-parameter models (Chinchilla ratio ≈ 50:1)
- BPE tokens carry semantic content; character tokens carry spelling
- Longer effective context per token → tests long-range transport

**Implementation**:
- HuggingFace `datasets` + `tokenizers` library for BPE
- Or use pre-tokenized WikiText-103 from HuggingFace
- ~2 hours to set up

### Optional: Synthetic Probes

Before scaling to full language modeling, synthetic tasks can isolate mechanism contributions:

| Task | What it tests | Expected winner |
|---|---|---|
| **Selective copy**: given key at pos t, output value from pos t' | Hopfield retrieval | Attention > Fiber > SSM |
| **Frequency discrimination**: classify by spectral content | Spectral representation | Spectral models > dense models |
| **Context-dependent recall**: meaning depends on distant context | Wilson line transport | Fiber > SSM (if phase rotation helps) |
| **Multi-scale pattern**: nested patterns at different scales | Subbundle structure | Spectral models (if subbundles learn scale separation) |

Each takes ~1 hour to build and ~30 min to run. They provide clean signal about whether specific mechanisms work in principle.

### Phase 2b: Wilson Fiber Is Decorative (2-3 days)

1. **Accept**: The geometric structure doesn't help for sequence modeling at this scale.
2. **Pivot to Option 3** (Parseval Attention) — test whether the geometry works as an attention metric instead of a recurrence.
3. **Or pivot to Option 4** (Fiber-Augmented Transformer) — see if the fiber adds anything to a working transformer.
4. **If both fail**: The spectral constellation representation itself may be the wrong abstraction. Consider whether the thesis findings point toward a different geometric primitive.

### Phase 3: Scale and Publish (1-2 weeks)

1. Whichever hybrid works, scale to 10M-50M params on full WikiText-103
2. Run proper scaling analysis (loss vs. compute curves)
3. Compare against Mamba-2, RetNet, RWKV baselines (not just GPT-Nano)
4. Write up for thesis chapter
